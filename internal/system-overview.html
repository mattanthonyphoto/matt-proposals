Tools Built

Five tools. One ecosystem.

site_audit.py

Crawls a website (up to 30 pages), checks SSL, meta tags, schema, sitemap, GBP, directories, alt text coverage, and content depth. Outputs scored JSON with pass/warn/fail findings.

generate_audit.py

Renders a visual SEO/GEO audit report from a JSON config. Dark-mode design with animated score rings, traffic-light findings, competitor tables.

generate_proposal.py

Scaffolds and renders Creative Partner proposals from JSON configs. Full brand design system with cinematic cover, animated roadmap, tier cards.

publish_proposal.py

Pushes HTML files to GitHub Pages. Handles git clone, commit, push. Returns live URL. Upserts on re-upload (same URL, new content).

Note: Supabase Storage was tested first but blocks HTML rendering (sandbox CSP policy). GitHub Pages serves HTML correctly and is free.

Key Learnings

What we learned building this.

Always run the real audit first. Summerhill was estimated at 28/100 SEO. Actual score: 85/100. The pitch would have lost credibility with inflated bad scores.

Competitor analysis must be local. Comparing a Gibsons builder to Whistler firms was wrong. The real competitors are Lincoln Construction (same town, already winning awards) and Jenkins Construction.

Adapt tiers to the prospect's situation. Summerhill already has a web team. Removing website work and lowering the entry point from $2,500 to $2,000 made the pitch more credible and accessible.

Lower tiers should be pure content. Blog drafts, SEO strategy, and awards pipeline belong in the top tier only. Foundation and Growth should be strictly content creation.

Supabase Storage can't host HTML. It forces content-type: text/plain and adds a sandbox CSP that blocks all CSS/JS. GitHub Pages works correctly and is free.
